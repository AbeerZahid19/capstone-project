\## Rules learned from vague vs precise prompting drill



1\. Always specify validation requirements explicitly (required fields, exact format rules like email regex) — vague prompts produce validation that looks correct but isn't verified.

2\. Every form/logic feature must include a verification step: write tests and run them before considering the feature done, not just visually inspect the code.

3\. Explicitly state accessibility requirements (labels, keyboard navigation, aria-live regions) in prompts — they are not included by default unless requested.
